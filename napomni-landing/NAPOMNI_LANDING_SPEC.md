# ТЗ на лендинг NAPOMNI — для Claude Code

## Что это
Лендинг-страница для продукта NAPOMNI — AI-ассистент «вторая память». Пользователь кидает в бота (Telegram / приложение) всё подряд — голос, текст, фото, файлы — а ИИ сам раскладывает по полочкам, напоминает когда нужно, и находит по запросу.

Цель лендинга: собрать заявки на ранний доступ (телефон или e-mail) и отправить их в Telegram-бот владельцу.

## Технические требования
- **Деплой**: Render (static site или Node.js Express для обработки формы)
- **Стек**: HTML + CSS + JS (один файл или минимум файлов). Можно использовать Express для бэкенда формы.
- **Форма**: при отправке заявки данные уходят через Telegram Bot API на указанный chat_id. Поля: имя (опционально), телефон (основное), e-mail (опционально). После отправки — сообщение «Спасибо! Вы в списке раннего доступа».
- **Telegram Bot API**: использовать fetch к `https://api.telegram.org/bot{TOKEN}/sendMessage`. TOKEN и CHAT_ID вынести в env-переменные.
- **Адаптивность**: идеально на мобильных (80% трафика из Telegram).
- **Лого**: файл `logo.png` в корне проекта. Логотип — скруглённый квадрат с градиентом синий→зелёный, белая галочка. Под ним текст NAPOMNI тёмно-синим.

## Дизайн-направление
**Стиль**: Premium tech product. Чистый, с воздухом, но не пустой. Как лендинги Notion, Linear, Arc Browser. Тёмная тема (тёмно-серый/почти чёрный фон #0A0A0F), яркие акценты из палитры лого (градиент #2196F3 → #4CAF50 → #CDDC39).

**Типографика**: Google Fonts — заголовки `Outfit` (bold, крупно), текст `DM Sans` (regular). Никакого Inter/Roboto/Arial.

**Ключевой приём**: на hero-секции — анимированный мокап телефона с интерфейсом Telegram-чата, где видно переписку с ботом NAPOMNI. Это сразу показывает суть продукта.

## Бэкенд (форма → Telegram)

Минимальный Express-сервер (Node.js):

```javascript
// server.js
const express = require('express');
const path = require('path');
const app = express();

app.use(express.json());
app.use(express.urlencoded({ extended: true }));
app.use(express.static('public'));

app.post('/api/subscribe', async (req, res) => {
  const { phone, email } = req.body;
  
  const text = `🆕 Новая заявка NAPOMNI!\n📱 Телефон: ${phone}\n📧 Email: ${email || 'не указан'}`;
  
  try {
    await fetch(`https://api.telegram.org/bot${process.env.TG_BOT_TOKEN}/sendMessage`, {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({
        chat_id: process.env.TG_CHAT_ID,
        text: text,
        parse_mode: 'HTML'
      })
    });
    res.json({ success: true });
  } catch (err) {
    process.stdout.write('Error: ' + err.message + '\n');
    res.status(500).json({ success: false });
  }
});

const PORT = process.env.PORT || 3000;
app.listen(PORT, () => process.stdout.write(`Server running on port ${PORT}\n`));
```

Env-переменные для Render:
- `TG_BOT_TOKEN` — токен Telegram-бота
- `TG_CHAT_ID` — chat_id Сергея для получения заявок

## Анимации
- Hero: сообщения в мокапе телефона появляются последовательно (каждое через 1.5с)
- Секции: fade-in при скролле (IntersectionObserver)
- Карточки возможностей: лёгкий hover-glow
- Кнопки: при наведении — градиент слегка сдвигается (background-position transition)
- Форма: после отправки — confetti-эффект (CSS-only, 5-6 кружочков разлетаются)

## Важно
- Все тексты на русском языке
- Мобильная версия — приоритет (большинство придёт из Telegram)
- Мокапы телефона и ноутбука — рисовать на CSS/SVG, не использовать внешние картинки
- Чат в мокапе должен выглядеть реалистично, как настоящий Telegram
- Использовать `process.stdout.write()` вместо `console.log()` для логирования
- Favicon: можно использовать тот же logo.png
- Meta-теги для шаринга: title «NAPOMNI — Помню всё за тебя», description «AI-ассистент, который запоминает всё и напоминает когда нужно. Кидай текст, фото, голос — он разберёт.»
